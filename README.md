# Personal Assistant Bot
Телеграм-бот для персональной автоматизации, который управляет расписаниями, предоставляет персональные подсказки и собирает данные через интеграции.

## Технологический стек
- Python 3.11+
- Aiogram 3
- AIOHTTP
- APScheduler
- Docker / Docker Compose
- PostgreSQL (ожидается через `utils.db` и `docker-compose.yml`)

## Функционал
- Асинхронная обработка Telegram-апдейтов и команд через `aiogram`.
- Персональные сценарии взаимодействия с пользователем (`handlers/personal.py`, `handlers/settings.py`).
- Поддержка административных и информационных команд (`handlers/admin.py`, `handlers/info.py`).
- Планировщик фоновых задач для напоминаний и уведомлений (`utils/scheduler.py`).
- Интеграция с внешними API и базой данных через модульные хелперы (`utils/api.py`, `utils/db.py`).
- Стандартизированная система логирования с уровнями DEBUG/INFO/WARNING/ERROR/CRITICAL/SUCCESS/EVENT/SYSTEM.
- Готовность к развёртыванию в Docker-среде.

## Установка и запуск проекта
1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/<your-account>/personal_assistant_bot.git
   cd personal_assistant_bot
   ```
2. **Создайте и заполните файл окружения**
   ```bash
   # Windows
   copy NUL .env
   
   # Linux/macOS
   touch .env
   ```
   Заполните переменные, перечисленные в `config.py` (например, `BOT_TOKEN`, `DATABASE_URL` и др.).
3. **Создайте виртуальное окружение и установите зависимости**
   ```bash
   python -m venv .venv
   .\.venv\Scripts\activate
   pip install --upgrade pip
   pip install -r requirements.txt
   ```
4. **Запустите бота локально**
   ```bash
   python bot.py
   ```
5. **Альтернативный запуск через Docker Compose**
   ```bash
   docker-compose up --build
   ```

## Архитектура проекта
- `bot.py` — точка входа, инициализация диспетчера, маршрутов и системных сервисов.
- `config.py` — загрузка конфигурации из окружения и настройка зависимостей.
- `handlers/` — набор роутеров `aiogram` для разных сценариев взаимодействия (администрирование, базовые команды, персональные диалоги).
- `utils/` — вспомогательные сервисы: работа с внешними API, базой данных, планировщиком задач и логгером.
- `docker-compose.yml` и `dockerfile` — контейнеризация приложения вместе с зависимыми сервисами.
- `logs/stream_monitor.log` — файл с логами приложения; создаётся автоматически при старте через встроенный логгер.
